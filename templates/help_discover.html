What is URL Discovery?

URL Discovery allows you to quickly map out all the pages on a website without running time-consuming accessibility checks. This is useful for:

  • Understanding website structure before scanning
  • Planning targeted accessibility scans
  • Creating site maps for documentation
  • Identifying the scope of a website
Key Difference: URL Discovery finds and lists pages but doesn't check them for accessibility issues. It's much faster than a full scan.

When to Use URL Discovery

Good Use Cases
  • Planning multi-section scans
  • Discovering site structure
  • Finding all pages in a subdirectory
  • Creating URL lists for manual review
  • Estimating scan time/scope
When to Scan Instead
  • You need accessibility violation reports
  • You want WCAG compliance checking
  • You need detailed issue descriptions
  • You're doing formal accessibility audits

How to Use URL Discovery

Step 1: Navigate to Discover Page

Click "Discover" in the navigation menu or go to /discover

Step 2: Enter Website URL

Enter the URL you want to explore. Include the full protocol (https://).

Example: https://example.com/section

Step 3: Set Depth Level

Choose how deep to crawl:

  • All Pages: Discover all reachable pages (comprehensive)
  • 1 Level Deep: Only pages linked directly from start URL
  • 2+ Levels: Follow links up to specified depth

Step 4: Start Discovery

Click "Discover URLs". The system will:

  1. Crawl the website following links
  2. List all discovered URLs
  3. Show total count and duration
  4. Display each URL with a "Scan" button

Step 5: Scan Individual URLs (Optional)

After discovery completes, each URL in the results list has a "Scan" button:

  • Click the "Scan" button next to any URL
  • You'll be taken to the scan page with that URL pre-filled
  • Adjust scan settings if needed, then start the accessibility scan
  • This allows you to quickly scan specific pages you discovered
Quick Workflow: Discover all pages → Click "Scan" on priority URLs → Get focused accessibility reports

Advanced Options

Same Domain Only

When enabled: Only discovers URLs on the same domain as the starting URL

Example: Starting at example.com won't follow links to external-site.com

Recommendation: Keep this enabled to focus on your website

Restrict to Path

When enabled: Only discovers URLs within the starting path

Example: Starting at example.com/blog won't discover example.com/products

Use case: Focus on specific website sections

Max Pages

What it does: Limits the number of URLs discovered

Default: 100 pages

Max: 10,000 pages (for very large sites)

Working with Discovered URLs

Scan Individual URLs

Each discovered URL has a "Scan" button for quick accessibility scanning:

  1. Review the list of discovered URLs
  2. Click the "Scan" button next to any URL you want to check
  3. The scan page opens with the URL pre-filled
  4. Configure scan settings (or use defaults) and start scanning
One-Click Scanning: No need to copy/paste URLs - click "Scan" and you're ready to go!

Export URLs for Planning

Download discovered URLs for analysis and documentation:

  • CSV: Open in Excel, Google Sheets for organization and analysis
  • JSON: Import into other tools or scripts
  • TXT: Simple list format, one URL per line
  • Copy: Copy selected URLs to clipboard for quick sharing
Multiple Formats: Choose the format that works best for your workflow and tools.

What You Can Do With Discovered URLs

  • Scan selectively: Use the "Scan" button to check high-priority pages
  • Plan targeted scans for specific sections
  • Share with developers for review
  • Create documentation of site structure
  • Identify pages that need priority scanning
  • Compare against expected site structure

Tips & Best Practices

  • Use "Scan" Buttons for Quick Checks: Click the "Scan" button next to any URL to immediately check it for accessibility issues
  • Start Broad, Then Narrow: First discover all pages, then use individual "Scan" buttons on priority sections
  • Check Large Sites in Sections: For 1000+ page sites, use path restriction to discover one section at a time
  • Document Your Site: Regular URL discovery helps track when new pages are added
  • Combine Discovery with Targeted Scans: Use discovery to find pages, then click "Scan" on important ones for detailed accessibility reports
  • Export for Planning: Download URLs to spreadsheet for organizing scan priorities with your team
